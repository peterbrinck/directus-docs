# Field Types

To support multiple SQL vendors (MySQL, PostgreSQL, MS-SQL, etc) Directus has a generalized set of field types to define API output (array, json, boolean, etc) and system data (m2o, alias, group, etc).

  • alias â€“ Fields that do not have corresponding column in the database
  • array â€“ Standard array format in API response
  • binary â€“ Binary strings limited by its length
  • boolean â€“ true or false
  • datetime â€“ A date and time in ISO format, eg: 2018-09-19T14:00:43+00:00
  • date â€“ Date, eg: 2018-09-19
  • time â€“ Time, eg: 14:09:22
  • file â€“ Foreign key to directus_files.id
  • group â€“ Groups fields together visually, children save group into directus_fields.group
  • integer â€“ Whole number
  • decimal â€“ Number that includes a decimal
  • json â€“ Standard JSON format in API response
  • lang â€“ Specific to translation interfaces, this stores the language key
  • m2o â€“ Many-to-One Relationship
  • o2m â€“ One-to-Many Relationship
  • sort â€“ System field that stores a item order within the collection items
  • status â€“ System field used for publishing workflows
  • string â€“ Any text or characters, limited by its length
  • translation â€“ Specific to translation interfaces, this is a o2m for multi-lingual content
  • uuid â€“ A Universally Unique Identifier
  • datetime_created â€“ System field to track the datetime an item was created, used by revisions
  • datetime_updated â€“ System field to track the datetime an item was updated, used by revisions
  • user_created â€“ System field to track the user who created an item, used by revisions
  • user_updated â€“ System field to track the user who updated an item, used by revisions

Datatypes

When creating or updating fields you can also set a vendor-specific datatype for more granular DBA control over data storage.
